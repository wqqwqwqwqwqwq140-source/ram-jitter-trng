RAM Jitter TRNG

Hardware True Random Number Generator in C++

The Origin of the Idea

Countless mathematical formulas exist to generate pseudo-random numbers. However, harvesting pure chaos directly from the underlying hardware requires a different approach, free of software layer abstractions.

"I conceived the core concept of this project when I was 12 years old. The idea was straightforward: push bits continuously along a CPU to RAM path and measure the elapsed time. The physics of the hardware inherently introduces true randomness."

Years later, this exact concept has been implemented in low-level C++ utilizing native assembly instructions.

Hardware Mechanics (CPU to RAM)

Due to thermal noise in microchips, memory bus contention, and periodic dynamic RAM (DRAM) refresh cycles, memory access times fluctuate slightly at a microscopic level.

The intrinsic __rdtsc() function records the exact number of CPU clock cycles before and after the data loop. The Least Significant Bit (LSB) of this execution delta varies unpredictably due to physical electronic noise.

Core C++ Implementation

// Measuring raw execution jitter
unsigned long long start = __rdtsc();

for (int i = 0; i < size; ++i) {
    buffer[i] = i ^ 0x55;           // Write to RAM
    volatile char read = buffer[i]; // Read from RAM
}

unsigned long long end = __rdtsc();
return (end - start) & 1;           // Extract the true random bit
